# 增车建议功能接口文档

## 1. 接口概述

增车建议功能是铁路客运数据分析系统中的运力优化模块，用于根据客流数据自动生成或人工添加列车增开建议，并支持审核流程管理。

## 2. 接口列表

### 2.1 获取增车建议（系统生成）

- **接口地址**：`GET /api/line-optimization/addition-suggestions`
- **功能描述**：基于过载告警数据和客流统计自动生成增车建议
- **请求参数**：无
- **请求示例**：`GET /api/line-optimization/addition-suggestions`

#### 返回数据示例
```json
{
  "code": 20000,
  "data": [
    {
      "lineCode": "G123",
      "section": "北京南-上海虹桥",
      "suggestedTrainNumber": "G9999",
      "departureTime": "08:30:00",
      "arrivalTime": "12:30:00",
      "carriageCount": 8,
      "trainType": "CR400AF",
      "reason": "该区间在8:00-9:00时段满载率高达95%，建议加开列车",
      "expectedLoadRate": 85.0,
      "status": "PENDING",
      "createdBy": "SYSTEM"
    }
  ],
  "message": "操作成功"
}
```

### 2.2 根据线路编码获取增车建议（分页）

- **接口地址**：`GET /api/line-optimization/addition-suggestions-by-line`
- **功能描述**：根据线路编码分页获取增车建议
- **请求参数**：
  - `lineCode` (可选, String): 线路编码
  - `current` (可选, Integer, 默认: 1): 页码
  - `size` (可选, Integer, 默认: 10): 每页大小
- **请求示例**：`GET /api/line-optimization/addition-suggestions-by-line?lineCode=G123&current=1&size=10`

#### 返回数据示例
```json
{
  "code": 20000,
  "data": {
    "records": [
      {
        "lineCode": "G123",
        "section": "北京南-上海虹桥",
        "suggestedTrainNumber": "G9999",
        "departureTime": "08:30:00",
        "arrivalTime": "12:30:00",
        "carriageCount": 8,
        "trainType": "CR400AF",
        "reason": "该区间在8:00-9:00时段满载率高达95%，建议加开列车",
        "expectedLoadRate": 85.0,
        "status": "PENDING",
        "createdBy": "SYSTEM"
      }
    ],
    "total": 1,
    "size": 10,
    "current": 1,

  "message": "操作成功"
}
```

### 2.3 添加人工增开建议

- **接口地址**：`POST /api/line-optimization/addition-suggestions`
- **功能描述**：添加人工增开建议
- **请求参数**：请求体为JSON格式
  - `lineCode` (String): 线路编码
  - `section` (String): 区间
  - `suggestedTrainNumber` (String): 建议车次号
  - `departureTime` (String): 建议发车时间
  - `arrivalTime` (String): 建议到达时间
  - `carriageCount` (Integer): 建议编组
  - `trainType` (String): 建议车型
  - `reason` (String): 建议原因
  - `expectedLoadRate` (Double): 预计满载率
- **请求示例**：
```json
{
  "lineCode": "G456",
  "section": "广州南-深圳北",
  "suggestedTrainNumber": "G8888",
  "departureTime": "14:00:00",
  "arrivalTime": "15:30:00",
  "carriageCount": 8,
  "trainType": "CR400BF",
  "reason": "春运期间该线路客流激增，需增加运力",
  "expectedLoadRate": 75.0
}
```

#### 返回数据示例
```json
{
  "code": 20000,
  "data": {
    "lineCode": "G456",
    "section": "广州南-深圳北",
    "suggestedTrainNumber": "G8888",
    "departureTime": "14:00:00",
    "arrivalTime": "15:30:00",
    "carriageCount": 8,
    "trainType": "CR400BF",

    "expectedLoadRate": 75.0,
    "status": "PENDING",

  },
  "message": "操作成功"
}
```

### 2.4 分页查询增车建议（组合条件查询）

- **接口地址**：`GET /api/line-optimization/addition-suggestions/paging`
- **功能描述**：根据线路、区间、状态等条件分页查询增车建议
- **请求参数**：
  - `lineCode` (可选, String): 线路编码
  - `section` (可选, String): 区间
  - `status` (可选, String): 建议状态 (PENDING/APPROVED/REJECTED)
  - `createdBy` (可选, String): 建议创建者类型 (SYSTEM/MANUAL)
  - `current` (可选, Integer, 默认: 1): 页码
  - `size` (可选, Integer, 默认: 10): 每页大小
- **请求示例**：`GET /api/line-optimization/addition-suggestions/paging?lineCode=G123&status=APPROVED&current=1&size=10`

## 3. 数据模型

### 3.1 TrainAdditionSuggestionVO
| 字段名 | 类型 | 描述 |
|--------|------|------|
| lineCode | String | 线路编码 |
| section | String | 需要加车的区间 |
| suggestedTrainNumber | String | 建议车次号 |
| departureTime | LocalTime | 建议发车时间 |
| arrivalTime | LocalTime | 建议到达时间 |
| carriageCount | Integer | 建议编组 |
| trainType | String | 建议车型 |
| reason | String | 建议原因 |
| expectedLoadRate | Double | 预计满载率 |
| status | String | 建议状态 (PENDING: 待处理, APPROVED: 已批准, REJECTED: 已拒绝) |
| createdBy | String | 建议创建者类型 (SYSTEM: 系统生成, MANUAL: 人工添加) |

## 4. 状态说明

- `PENDING`: 待处理 - 建议刚生成或添加，等待审核
- `APPROVED`: 已批准 - 建议已通过审核，可执行
- `REJECTED`: 已拒绝 - 建议被拒绝，不执行

## 5. 使用说明

1. **系统自动生成建议**：系统会根据过载告警和客流数据自动生成增车建议，并将状态设为 `PENDING`
2. **人工添加建议**：用户可通过 POST 接口手动添加增车建议，状态默认为 `PENDING`
3. **分页查询**：可按线路、区间、状态等条件进行组合查询
4. **审核管理**：建议生成后需要进行审核，更新状态为 `APPROVED` 或 `REJECTED`

## 6. 错误码说明

- `20000`: 操作成功
- `40000`: 请求参数格式有误
- `40400`: 数据不存在
- `40100`: 未通过认证
- `40300`: 禁止访问
- `99999`: 未知错误
